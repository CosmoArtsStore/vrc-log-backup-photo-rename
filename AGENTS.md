# STELLAProject Agent Rules

## 重大規約

- 常に毎回のセッションで必ず質問を挟まず、`npm run tauri build`を権限昇格で実行すること。（`vite` の `esbuild` 起動が `spawn EPERM` で止まるため）タイムアウトで止まった場合も確認を挟まず十分な待ち時間へ自動で延長して再実行してよい。
- ファイル読み込み時は Get-Content -Encoding UTF8 を使用すること。
- 共通規約 `.agents/rules/coding-standard.md` を全Project共通の正規規約として扱うこと。

## コード実装規約
- 業界標準のライブラリ、実装方式、データ形式、設計パターンで補える処理は原則として独自実装しないこと
- 独自実装は「業界標準では要件を満たせない」ことを確認できた場合に限り許可し、その理由と採用しない標準案を`F:\DEVELOPFOLDER\STELLAProject\UserTools\reports\report.md`へ残すこと
- チーム開発、保守、第三者レビューに耐えることを前提に実装判断すること
- エラーハンドリングは文脈付きメッセージを原則とすること
- 継続可能な失敗はログを残し、失敗の黙殺を避けること。例外を残す場合は理由コメントを必須とすること。

## Polaris要件
- ユーザーデータと他アプリの実ファイルへ直接触れる高リスク領域として扱うこと
- 規約準拠に加えて「まず理解できる設計」を優先し、過剰な抽象化を避けること
- 変更では「ユーザーデータを壊さない」を最低条件として確認すること
