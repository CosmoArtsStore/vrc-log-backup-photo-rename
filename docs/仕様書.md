## 1. 概要

本仕様書は、VRChatのログファイルを自動バックアップするPCアプリケーション「LogBackupTool」の設計仕様を定義するものである。本ツールはインストーラーで導入され、複数のコンポーネントにより構成される常駐型バックアップシステムである。

### 1.1 コンポーネント一覧

| コンポーネント名 | 種別 | 実装 | 概要 |
|---|---|---|---|
| OnsiteLogBackupTool.exe | 常駐アプリ（バックエンド） | Rust Binary | ログバックアップを自動実行するメインプロセス（タスクトレイ常駐） |
| LBTAppObserver.exe | 常駐アプリ（モニタリング） | Rust Binary | OnsiteLogBackupTool の動作状態をロギングするCLIモニタリングツール |
| CAS_LBTSetting.exe | 設定GUIアプリ | Tauri + React | Config.json の編集・手動バックアップ・スタートアップ設定を行うGUIアプリ |
| lbt_core | 共通ライブラリ | Rust Library | Config.json・appinfo.log の読み書き等、共通ロジックを集約 |
| Config.json | 設定ファイル | — | OnsiteLogBackupTool の動作を制御するJSON設定ファイル |
| appinfo.log | ログファイル | — | OnsiteLogBackupTool が出力するランタイムログ（起動毎に上書き） |
| NSIS インストーラー | インストーラー | NSIS Script | 全コンポーネントのインストール・アンインストール・レジストリ管理 |

### 1.2 リポジトリ構成

```
f:\DEVELOPFOLDER\RE-NAME-SYS\LogBackupTool\
├── Cargo.toml                     ← Workspace Root
├── cas_lbt_setting/               ← Tauri App (設定GUI)

│   └── package.json

├── lbt_app_observer/              ← Rust Binary (モニタリング)
```

## 3. インストーラー仕様（NSIS）

### 3.1 インストール処理フロー

1. **既存インストールの検出**
   - インストール先フォルダの存在を確認する

2. **既存インストールが存在する場合**
   - 常駐プロセス（OnsiteLogBackupTool.exe、LBTAppObserver.exe）へ `WM_CLOSE` 等のメッセージ送信による正常終了を試みる。`taskkill` による強制終了は行わない（OSによる意図しない再起動トリガーの発動を抑制するため）
   - `Config.json` を除く、インストールフォルダ内のすべてのファイルおよびサブフォルダを削除する

3. **クリーンインストールの実施**
   - 各実行ファイルおよび Config.json を所定のフォルダへ配置する
   - インストール先は `$LOCALAPPDATA\CosmoArtsStore\LogBackupTool` に強制展開する
   - `appinfo.log` は OnsiteLogBackupTool.exe 起動時に自動生成されるため、インストーラーは生成しない

4. **レジストリ登録（初期セットアップ）**
   - インストール完了時に `HKCU\...\Run` キーへ OnsiteLogBackupTool.exe・LBTAppObserver.exe のフルパスを物理的に書き込み、初回起動時からの自動起動を確定させる（初期値: ON）
   - 登録先: `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`

### 3.2 アンインストール処理

- 常駐プロセスへ `WM_CLOSE` 送信により正常終了させる
- レジストリ内のスタートアップ登録値（両exe分）を完全に削除し、OS側に不要なエントリを残さない
- インストールフォルダ全体を削除する

> ※ ユーザーデータ（BackupFile 配下）はアンインストール対象外とし、削除しない。

### 3.3 インストーラー実装方針

- Tauri ビルトインの NSIS スクリプトを拡張するか、カスタム NSIS スクリプトで3つのexeを1つのインストーラーに統合する。

## 4. OnsiteLogBackupTool.exe 詳細仕様

### 4.1 概要

PCに常駐し、VRChatのログファイルを自動バックアップするバックエンドアプリケーション。ウィンドウは表示せず、タスクトレイに常駐する。

| 項目 | 内容 |
|---|---|
| 実装言語 | Rust |
| サブシステム | `windows_subsystem = "windows"`（コンソール非表示） |
| 主要クレート | `tray-icon`（タスクトレイ）、`sysinfo`（プロセス監視） |

### 4.2 動作フロー

#### ① 起動時 — Config.json の読み込み

- 起動時（初回のみ）`Config.json` を一時ファイル（tmp）としてコピーした後、`lbt_core::config` 経由で読み込む。

#### ② 異常終了トリガーの担保

- `RegisterApplicationRestart` Windows API を起動時に呼び出し、プロセスが異常終了した際に自動再起動されるようOSへ登録する。
- 正常終了（タスクトレイ等からの終了）では再起動は行われない。

#### ③ 起動時バックアップ

- 以下のパス配下に存在する `output_log` ファイルを検索し、起動時の初回バックアップを実施する。

```
%APPDATA%\..\LocalLow\VRChat\VRChat\
```

#### ④ ディレクトリ容量監視

- 以下の2ディレクトリの容量を調査する。
  - `%LOCALAPPDATA%\CosmoArtsStore\LogBackupTool\BackupFile`（デフォルトバックアップ先）
  - CAS_LBTSetting.exe でユーザーが指定したカスタムディレクトリ（存在する場合）
- 容量の閾値は `Config.json` 内の `capacityThresholdBytes`（デフォルト: 10GB = `10737418240` bytes）に基づく。
- 閾値超過時は警告を発行し、内容を `lbt_core::logger` 経由で `appinfo.log` へ出力する。
- 容量監視は **起動時** および **各バックアップ処理（終了時・手動）の実行直後** に実施する。

#### ⑤ VRChat 起動監視・終了時バックアップ

- `sysinfo` クレートを用い、5秒間隔で `vrchat.exe` のプロセス監視ポーリングを行う。
- `vrchat.exe` の起動を検知した場合、「VRC起動中」フラグを立てる。
- プロセスの停止または存在しないことを確認した場合、バックアップを実施する。
- バックアップ対象: `%APPDATA%\..\LocalLow\VRChat\VRChat\` 配下の `output_log*.txt`

### 4.3 appinfo.log の仕様

| 項目 | 内容 |
|---|---|
| 出力タイミング | 各バックアップ実行時、容量警告発生時、起動・終了時 |
| ファイル管理 | 起動毎に上書き（`Truncate` モードで開く）。前回ログは保持されない |
| 出力フォーマット | タイムスタンプ + メッセージ（`lbt_core::logger` の共通関数で統一） |
| 用途 | LBTAppObserver.exe によるリアルタイム表示用 |

### 4.4 タスクトレイ

- `tray-icon` クレートを使用してシステムトレイにアイコンを表示する。
- 右クリックメニューに「終了」を設置し、選択時は OS 再起動予約を解除してプロセスを正常終了する。

### 4.5 異常終了・再起動

| 終了種別 | OS再起動 | 備考 |
|---|---|---|
| 異常終了（クラッシュ等） | あり | `RegisterApplicationRestart` によりOSが自動再起動 |
| 正常終了（タスクトレイ） | なし | ユーザーの意思による停止 |

> ※ スタートアップからの無効化手順をマニュアルに必ず記載し、ユーザーが自分の意思でバックアップを停止できることを明示すること。

## 5. LBTAppObserver.exe 詳細仕様

### 5.1 概要

OnsiteLogBackupTool.exe の動作状態をウィンドウにロギング表示するCLIアプリケーション。OnsiteLogBackupTool.exe が出力した `appinfo.log` を取り込み、内容をリアルタイムで表示する。

| 項目 | 内容 |
|---|---|
| 実装言語 | Rust |
| サブシステム | `windows_subsystem = "console"`（コンソールウィンドウ表示） |

### 5.2 動作仕様

| 項目 | 仕様 |
|---|---|
| 監視間隔 | 5秒 |
| 監視対象 | `appinfo.log`（`%LOCALAPPDATA%\CosmoArtsStore\LogBackupTool\Backend\appinfo.log`） |
| 表示方式 | `tail -f` 相当のループ実装。5秒毎にファイル末尾までの新規行を取得してコンソールへ出力 |
| ファイルオープンモード | **共有読込モード（ReadOnly / ShareRead）** で開く。OSレベルの共有違反を回避するため必須 |
| 終了方法 | コンソールウィンドウを閉じる |
| 再起動プロセス | なし |
| 主な役割 | OnsiteLogBackupTool.exe の挙動を可視化するサポートアプリ。バックアップ処理自体は行わない |

## 6. CAS_LBTSetting.exe 詳細仕様

### 6.1 概要

`Config.json` を編集するGUIアプリケーション。設定の閲覧・変更、手動バックアップの実行、スタートアップ設定を提供する。

| 項目 | 内容 |
|---|---|
| 実装 | Tauri（Rust バックエンド + React/Vite フロントエンド） |

### 6.2 Tauri コマンド一覧

| コマンド名 | 処理内容 |
|---|---|
| `get_config` | `lbt_core::config` 経由で Config.json を読み込み、フロントエンドへ返す |
| `save_config` | 受け取った設定値を Config.json へ書き込む。保存前に最新の Config.json を再読み込みして競合を防止する |
| `execute_manual_backup` | OnsiteLogBackupTool.exe の未起動・VRChatの未起動を確認後、バックアップを実行する |

### 6.3 Config.json の読み書き

- 起動時に `Config.json` を読み込み、現在の設定値をUIに反映する。
- 保存操作時にも最新の `Config.json` を再読み込みした上で更新を書き込む（競合防止）。
- 設定保存後は以下のカスタムポップアップを表示する。

> **ポップアップメッセージ**  
> 「設定はバックエンドアプリには即時反映されません。即時反映する場合、タスクトレイからアプリを再起動してください。」

### 6.4 手動バックアップ機能

「手動バックアップ」ボタン押下時、Tauri コマンド `execute_manual_backup` が以下の事前チェックを実施する。

- OnsiteLogBackupTool.exe が起動していないこと
- vrchat.exe が起動していないこと（VRChatが終了済みであること）

両条件を満たす場合のみバックアップを実行する。いずれかの条件を満たさない場合はユーザーへエラーを通知する。

### 6.5 スタートアップ設定

| 設定値 | 動作 | レジストリ操作 |
|---|---|---|
| ON（初期値） | PC起動時にOnsiteLogBackupTool.exe・LBTAppObserver.exeを自動起動 | `HKCU\...\Run` に両exeを登録 |
| OFF | PC起動時の自動起動を無効化 | `HKCU\...\Run` から両exeを削除 |

- 起動時にレジストリを参照し、登録状況に応じてUIの初期値を設定する。
- `enableStartup` フラグの変更は保存時に即時レジストリへ反映する。

> ※ OFFにするとバックアップが自動実行されないことをUI上に明記すること。

### 6.6 フロントエンド UI 要素

| UI要素 | 対応設定 / 処理 |
|---|---|
| バックアップ先パス入力 | `backupDestinationPath` |
| 容量警告閾値入力（GB換算表示） | `capacityThresholdBytes`（内部はbyte換算して保存） |
| スタートアップON/OFFトグル | `enableStartup` + レジストリ制御 |
| 手動バックアップボタン | `execute_manual_backup` コマンド呼び出し |

## 7. lbt_core 共通モジュール仕様

### 7.1 概要

OnsiteLogBackupTool・LBTAppObserver・CAS_LBTSetting の3つのコンポーネントが共有する Rust ライブラリ。Config.json とappinfo.log の読み書きロジックを一元管理する。

### 7.2 モジュール構成

| モジュール | ファイル | 役割 |
|---|---|---|
| config | `src/config.rs` | Config.json の読み込み・書き込み・バリデーション |
| logger | `src/logger.rs` | appinfo.log への書き込み（タイムスタンプ付きフォーマット統一） |

### 7.3 Config 構造体（案）

```rust
pub struct Config {
    pub backup_destination_path: String,   // バックアップ先（空 = デフォルトパス使用）
    pub capacity_threshold_bytes: u64,     // 容量警告閾値（bytes）
    pub enable_startup: bool,              // スタートアップ自動起動設定
}
```

### 7.4 ログ出力フォーマット

```
[YYYY-MM-DD HH:MM:SS] <メッセージ>
```

## 8. Config.json 仕様

### 8.1 概要

OnsiteLogBackupTool.exe の動作パラメータを定義するJSON設定ファイル。インストール時に初期値で生成され、CAS_LBTSetting.exe を通じてユーザーが編集する。

- **パス:** `%LOCALAPPDATA%\CosmoArtsStore\LogBackupTool\Backend\Config.json`

### 8.2 設定項目

| キー名 | 型 | 説明 | 初期値 |
|---|---|---|---|
| `backupDestinationPath` | string | バックアップ先フォルダパス（空の場合はデフォルトパスを使用） | `""` |
| `capacityThresholdBytes` | number (u64) | 容量警告閾値（バイト単位） | `10737418240`（10GB） |
| `enableStartup` | boolean | スタートアップ自動起動設定 | `true` |

> ※ 上記はドラフト案。実装時に要確認・追記。

### 8.3 サンプル

```json
{
  "backupDestinationPath": "",
  "capacityThresholdBytes": 10737418240,
  "enableStartup": true
}
```

## 9. バックアップ仕様

### 9.1 バックアップ対象

| 対象ファイル | パス |
|---|---|
| output_log（VRChatログ） | `%APPDATA%\..\LocalLow\VRChat\VRChat\output_log*.txt` |

### 9.2 バックアップ先

- **デフォルト:** `%LOCALAPPDATA%\CosmoArtsStore\LogBackupTool\BackupFile\`
- **カスタム:** CAS_LBTSetting.exe でユーザーが指定したパス（`backupDestinationPath` に保存）

### 9.3 バックアップ実行タイミング

| タイミング | トリガー | 実行者 |
|---|---|---|
| 起動時バックアップ | OnsiteLogBackupTool.exe 起動 | OnsiteLogBackupTool.exe |
| 終了時バックアップ | vrchat.exe プロセスの停止検知 | OnsiteLogBackupTool.exe |
| 手動バックアップ | CAS_LBTSetting.exe のボタン押下 | CAS_LBTSetting.exe（Tauriコマンド経由） |

### 9.4 ファイル命名規則と重複排除

**命名規則**

バックアップファイルは以下の形式で保存する。

```
output_log_YYYYMMDD_HHMMSS.txt
```

**重複排除ロジック**

バックアップ実行時、保存先に同名かつ **ファイルサイズが同一** のファイルが存在する場合は、差分なしとみなし処理をスキップする。ファイルサイズが異なる場合のみ上書きまたは新規保存を行う。

## 11. 制約・注意事項

### 11.1 設定反映タイミング

- CAS_LBTSetting.exe で変更した設定は、OnsiteLogBackupTool.exe には即時反映されない。
- 即時反映するには、タスクトレイから OnsiteLogBackupTool.exe を再起動すること。

### 11.2 ユーザーへの周知事項（マニュアル記載必須）

- スタートアップからの無効化手順を必ずマニュアルに記載すること。
- スタートアップをOFFにするとバックアップが自動実行されないことを明記すること。
- 手動バックアップ実行前にVRChatを終了する必要があることを明記すること。

### 11.3 appinfo.log の取り扱い

- OnsiteLogBackupTool.exe は起動毎に `appinfo.log` を上書き（`Truncate`）する。前回ログは保持されない。
- 過去ログの永続保存が必要な場合は、別途ログローテーション機構の検討が必要。

### 11.4 クリーンインストールの挙動

- 既存インストールが検出された場合、`Config.json` を除くすべてのファイルが削除される。
- `Config.json` は引き継がれるため、ユーザーの設定はインストール後も維持される。

### 11.5 実装上の詳細ルール

- **Config.json の tmp コピー目的:** `CAS_LBTSetting.exe` による書き込み中と `OnsiteLogBackupTool.exe` の読み込みが衝突し、共有違反（Error）でクラッシュすることを防ぐためのガードとして機能させる。

- **容量監視の実施タイミング:** 起動時に加え、各バックアップ処理（終了時・手動）の実行直後にもディレクトリ調査を実施し、常に最新の容量状態を監視する。

- **クラッシュループ対策:** `RegisterApplicationRestart` による再起動は、Windows OS の仕様に基づき「起動から60秒以上経過してからのクラッシュ」に対して有効となるよう、初期化処理の順序を調整する。

- **appinfo.log の競合回避:** `LBTAppObserver.exe` がログを読み取る際、OSレベルの共有違反を避けるため「共有読込モード（ReadOnly / ShareRead）」でファイルを開くことを必須とする。

- **インストーラーによるプロセス停止:** 既存インストールの更新時、常駐プロセスの停止は `taskkill` による強制終了ではなく、可能な限り `WM_CLOSE` 等のメッセージ送信による正常終了を試み、OSによる意図しない再起動トリガーの発動を抑制する。
