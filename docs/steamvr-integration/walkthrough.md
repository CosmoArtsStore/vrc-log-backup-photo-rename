# SteamVR連動バックアップ機能およびアーキテクチャ刷新の完了

ユーザーからの新しい要件に基づき、「バックアップ用コンソールアプリ（LogBackUpTool）」と「リネーム用GUIアプリ（PhotoReNameApp）」を分離し、より堅牢なSteamVR連動バックアップ環境を構築しました。

## 完了した主な変更項目

### 1. アプリの分離・マルチバイナリ構成
- 1つのTauriプロジェクトから `LogBackUpTool.exe` と `PhotoReNameApp.exe` の2つのバイナリを同時生成するよう構成変更しました。
- インストーラー実行時、どちらも指定のディレクトリへ正しく展開されます。

### 2. LogBackUpTool (常駐バックアップツール) の実装
- **永続的監視ループ**: SteamVRが終了するまでVRChatの起動と終了を何度でも監視し続ける常駐型として構築。
- **リソース最適化**: 監視ループに5秒間のスリープ（`sysinfo`利用時の負荷軽減）を挟み、VRプレイへのパフォーマンス影響を排除。
- **ファイルロック対策**: VRChat終了検知後、OSのファイルハンドル解放を待つため3〜5秒間の猶予を持たせました。さらに、失敗した場合は指数バックオフ（徐々に待機時間を延ばす）によるリトライを行います。

### 3. インストーラーとSteamVR登録の強化
- **インストール先の強制変更**: デフォルトの `Program Files` から変更し、`%LOCALAPPDATA%\CosmoArtsStore\LogBackUpTool` の直下に強制展開するようNSISフック (`hooks.nsi`) を実装しました。
- **SteamVR自動登録（LogBackUpTool側）**: PhotoReNameAppを起動した際（またはインストール時のフック実行時）に、マニフェスト生成のターゲットを `LogBackUpTool.exe` として書き出し、`vrpathreg.exe` を使って設定します。
- **完全なクリーンアンインストール**: アンインストール実行直後（マニフェスト解除後）に、手動で作成したログフォルダなども含めて親ディレクトリ自体を `RMDir /r` によって一網打尽に削除する仕組みを導入し、ゴミを残しません。

## 検証と動作確認の手順

ビルドされたインストーラー `target\release\bundle\nsis\PhotoReNameApp_0.1.0_x64-setup.exe` を用いて、以下の流れをご確認ください。

1. **インストール**
    インストーラー実行後、ファイルが `AppData\Local\CosmoArtsStore\LogBackUpTool` とその配下（`LogBackUp`フォルダなど）に配置されているか確認します。
2. **SteamVR連動の確認**
    SteamVRを起動し、しばらくすると `LogBackUpTool.exe` （コンソール画面）が自動的に立ち上がることを確認します。
3. **バックアップとリネームの動作確認**
    - コンソールが監視待機状態のまま、VRChatを起動・終了します。
    - コンソールに「ファイルロック解放待機...」等のメッセージが出た後、直近のログが指定バックアップフォルダにコピーされることを確認してください。
    - コンソールはそのまま次の起動待機に入ります。
4. **アンインストール**
    コントロールパネルからアンインストールした際、SteamVRのリストから消えると共に `AppData\Local\CosmoArtsStore\LogBackUpTool` フォルダそのものが綺麗に消去されるか確認します。
