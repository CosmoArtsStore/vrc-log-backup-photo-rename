# 対応済み確認指示書

以下の項目は既に対応指示が出ており、対応済みのはずである。
**各項目について実際にファイルを開いて確認し、対応済みか未対応かを報告すること。**
推測で「対応済み」と報告しないこと。必ずコードを確認してから答えること。

## 確認1: ログファイル名の統一（info.log）

全アプリで `error_info.log` / `crash.log` の参照が `info.log` に変更されているか。

確認コマンド:
```bash
grep -rn "error_info.log" .
grep -rn "crash.log" .
```

報告内容:
- ヒットした箇所があれば全て列挙する
- 全てのヒットが `info.log` に置き換わっていれば「対応済み」と報告する

**推奨対応:** 現状維持。

#### 確認2: println! / eprintln! の削除
**状態:** ✅ 対応済み
**詳細:** 全Rustファイルに対し `println!` / `eprintln!` を検索しましたが、ヒットはありませんでした。標準出力によるログ出力は完全に排除され、`log_warn` / `log_err` 等の専用関数に移行されています。
**推奨対応:** 現状維持。

#### 確認3: rust-toolchain.toml のバージョン
**状態:** ✅ 対応済み
**詳細:**

#### 確認5: 環境変数フォールバックの修正

**詳細:** `unwrap_or_default` / `unwrap_or_else` / `unwrap_or("")` を検索しましたが、環境変数の取得 (`std::env::var`) に直接チェーンして利用されている箇所はありませんでした。これらはエラー伝播 (`?`) または明示的なパターンマッチングによるエラーハンドリングに置き換わっています。
**推奨対応:** 現状維持。

#### 確認6: Asset Protocol スコープ（Alpheratz）
**状態:** ✅ 対応済み
**詳細:** `Alpheratz/src-tauri/tauri.conf.json` の設定を確認しました。
```json
"scope": [
  "**/*.png",
  "**/*.jpg",
  "**/*.jpeg",
  "**/*.webp",
  "**/*.psd",
  "**/*.tiff",
  "**/*.tif",
  "**/*.bmp",
  "**/*.gif"
```

**推奨対応:** 現状維持。

#### 確認7: Capabilities のネットワーク通信
**状態:** ✅ 対応済み
**詳細:** `capabilities/` 内の JSON ファイルおよび全 JSON ファイルに対し `network` / `http:` を検索しましたが、権限設定に関連するヒットはありませんでした。不要なネットワーク通信権限は削除されています。
**推奨対応:** 現状維持。

#### 確認8: NSIS taskkill の修正（StellaRecord・Alpheratz）
**状態:** ❌ 未対応
**詳細:** 
- `Polaris/src-tauri/windows/hooks.nsi`: `ExecWait "taskkill /F /IM Polaris.exe"`
- `StellaRecord/.../hooks.nsi`: `nsExec::Exec 'taskkill /F /IM StellaRecord.exe 2>nul'`
- `Alpheratz/.../hooks.nsi`: `nsExec::Exec 'taskkill /F /IM Alpheratz.exe 2>nul'`
Polaris は `ExecWait` を使用していますが、他 2 アプリは `nsExec::Exec` を使用しており、書き方が統一されていません。また、Polaris には `2>nul`（標準エラー出力の抑制）もありません。
**未対応の場合の該当箇所:** `f:\DEVELOPFOLDER\STELLARECORD\Polaris\src-tauri\windows\hooks.nsi:2,10`
**推奨対応:** 全て `nsExec::Exec 'taskkill /F /IM {EXE} 2>nul'` に統一。

#### 確認9: スタートアップ登録の削除（StellaRecord・Alpheratz）
**状態:** ✅ 対応済み
**詳細:** 
- `Polaris`: レジストリ `Run` キーへの書き込みあり（仕様通り）
- `StellaRecord`, `Alpheratz`: `hooks.nsi` 内で「スタートアップ自動登録なし」と明記されており、コードも存在しません。
**推奨対応:** 現状維持。

#### 確認10: Vite の sync スタイル統一

**推奨対応:** 現状維持。

#### 確認12: レジストリキーのパス（STELLAProject）
