# STELLARECORD Ecosystem — 現状の問題点と課題まとめ (2026-03-10)

全プロジェクトの調査結果に基づき、未対応の課題および新たに発見された不整合を以下にまとめる。
※ Miraは本調査の対象外。

---

## 🛑 最優先課題 (リリース・動作不能リスク)

### 1. アプリ間連携の完全な断絶（パス不整合の深刻化） (#1, #25)
当初の想定以上に、PolarisとStellaRecordの間で「書き出すパス」と「読み取るパス」が一致していません。
- **ログファイルのズレ:**
    - Polaris: `%LOCALAPPDATA%\CosmoArtsStore\polaris\error_info.log` に出力。
    - StellaRecord: `%LOCALAPPDATA%\CosmoArtsStore\STELLARECORD\Polaris\polaris_appinfo.log` を読み取ろうとしている。
- **アーカイブフォルダのズレ:**
    - Polaris: `%LOCALAPPDATA%\CosmoArtsStore\polaris\archive` に保存。
    - StellaRecord: `%LOCALAPPDATA%\CosmoArtsStore\STELLARECORD\Polaris\archive` から読み取ろうとしている。
- **期待される修正:** 全アプリの `config.rs` および `main.rs` で、ベースパスを `%LOCALAPPDATA%\CosmoArtsStore\stellarecord\...` 等の統一パスに揃える必要がある。

### 2. インストーラー(NSIS)の不備 (#15)
- **問題:** 全アプリの `hooks.nsi` にて `taskkill` が実行されているが、対象がない場合にエラーメッセージが表示され、コマンドプロンプトのウィンドウが一瞬ポップアップする。
- **修正内容:** `nsExec::Exec` または `/F /IM ... 2>nul` を使用し、非表示かつエラー抑制状態で実行するように修正する。

### 3. モノレポ構造・スクリプトの形骸化 (#11)
- **ルート package.json:** `workspaces` 定義が欠落しており、一括管理ができていない。
- **StellaRecord/package.json:** 存在しない `pleiades_alpheratz` ディレクトリへの参照が残っている。
- **修正内容:** 正しいディレクトリ構造に基づき、ワークスペースとスクリプトを再定義する。

## 🟠 高・中優先度 (リリース品質・保守性)

### 4. 環境変数取得の脆弱性 (#20)
- **問題:** `Alpheratz` 等で `USERPROFILE` を `unwrap_or_else` でカレントディレクトリにフォールバックしている箇所がある。
- **修正:** 重要な環境変数が取得できない場合は、処理を中断してエラーを出す（サイレントに不正なパスで作動させない）。

### 5. Alpheratz の不完全なロギング (#25)
- **問題:** 依然として `println!` がスキャンロジック等に残っており、リリースビルドで動作状況を確認する手段がない。

### 6. Asset Protocol スコープ・Capabilities の超過許可 (#17, #22)
- **問題:** `Alpheratz` のスコープが `["**"]` のまま。
- **修正:** 拡張子制限と、実行時動的許可への移行。また全アプリで不要な Capabilities (shell, fs 等の広範な許可) を絞り込む。

### 7. Rust Toolchain バージョンの非固定 (#24)
- **現状:** `rust-toolchain.toml` が `channel = "stable"` になっており、環境によりビルド結果が変わりうる。
- **修正:** `1.77.0` 等の特定バージョンに固定する。

## 🟡 低・ポリッシュ

### 8. TSConfig 構造の不一致 (#13)
- `Alpheratz` と `StellaRecord` でプロジェクト参照形式（references）かどうかが分かれており、開発環境の設定が不統一。

### 9. NSIS アセット形式の不備 (#7)
- `Alpheratz` の `headerImage` 等が `.bmp` ではなく `.png` を指定している。

### 10. ウィンドウタイトルの不一致 (#8)
- `StellaRecord` のウィンドウタイトルが `STELLA_RECORD` (スネークケース) のまま。

---

## 完了済みまたは現状維持の項目
- **#1 アプリケーション識別子の衝突:** Polaris, StellaRecord, Alpheratz で個別の ID が設定済み。
- **#4 二重起動防止:** StellaRecord, Polaris ともに Mutex によるロックを実装済み。
- **#18 Polarisの構成:** Tauri 構造を維持したまま Rust メインで動作。現状維持。
- **#19 Panic Hook:** 全アプリに実装済み。
- **#21 Windows API:** 古い `winapi` への直接依存は排除され、`windows` クレートに移行済み。
