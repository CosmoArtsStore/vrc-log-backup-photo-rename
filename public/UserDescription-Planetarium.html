1. パースエンジンの詳細

文字列から「体験」を復元する

Planetarium は VRChat のカオスなログから特定のパターンを抽出します。特に Unity Rich Text タグが含まれる特殊なログ（USharpVideo等）も考慮した堅牢な正規表現を使用しています。

主要な抽出トリガー

  • Entering Room: ワールド訪問の開始（インスタンス情報含む）
  • OnPlayerJoined: 他者との接触。usr_ID の追跡（モード依存）
  • Switching Avatar: 誰がいつ何に着替えたかの記録
  • Video Playback: どの動画URLがリクエストされたか
抽出パターン例 (Regex)
// 動画再生イベントの抽出
\[USharpVideo\] Started video load
for URL: (.*?), requested by (.*)

※ タグ除去処理を含む 2-stage パースを実施

2. データベース構造 (planetarium.db)

リレーショナルな思い出管理

単なるテキストの保存ではなく、セクション（起動単位）、ビジット（ワールド単位）、プレイヤーの多対多関係を正確にモデル化しています。

テーブル記録単位
app_sessionsVRChat1回の起動～終了
world_visits1つのワールドへの滞在
playersプレイヤーマスタ（表示名/ID）
player_visits誰がどのワールドに居たか
整合性チェック
重複インポート防止 有効
外部キー制約 有効
WALモード 有効

3. プライバシー保護設計

「他者を監視しない」という構造的証明

enableUserTracking 設定により、DBに保存するデータの粒度を制御します。デフォルトの Privacy モードでは、他者の `usr_ID` を一切保存しません。

Privacy モード時の制限

  • players.user_id は常に NULL
  • 表示名のみを記録し、特定個人の追跡を物理的に不可能化
  • インスタンスオーナー情報の保存をスキップ
Safety Status
API Access DISABLE (No API)
Tracking Capability OPT-IN ONLY

4. アーカイブ処理 (zstd)

100年後も読み返せるポータビリティ

DBへのインポートが完了した生ログは、そのまま放置せず tar.zst 形式で圧縮保存されます。これにより、ストレージ容量を節約しつつ「生データ」も保全します。

圧縮仕様

  • アルゴリズム: Zstandard (zstd)
  • コンテナ: tar
  • 命名規則: YYYYMMDD_HHMMSS.tar.zst
ストレージ効率
生ログ (100MB) 100%
zstd 圧縮後 約 5% - 8%
