基本的な流れ

本アプリの動作について

Polaris は、ログ保全のために バックグラウンドで常駐して動作することを前提としたツール です。
PC起動時に自動で開始されますが、CPUやメモリの使用量はほぼ「ゼロ」であり、PCの動作やゲームのパフォーマンスを阻害することはありません。

1. 自動起動

PCを起動したら、すでに動いています

Polaris はデスクトップ上に何も表示しません。PC にログインした瞬間からバックグラウンドで常駐し、VRChat の起動を待ち続けます。

起動直後のアクション

  • Windows に「再起動してほしい」とお願いを登録する
  • VRChat がすでに起動していないかを確認する
  • VRChat が閉じていれば → 前回のログをすぐに同期
  • VRChat が開いていれば → 終了するまで静かに待機する
起動直後の動作イメージ
Polaris 常駐 稼働中
VRChat 確認 未起動を確認
初回同期 完了

2. VRChat 監視

3秒ごとに、静かに見守る

Polaris は3秒おきにパソコンで動いているアプリの一覧を確認し、VRChat がいるかどうかだけをチェックします。それ以外のことは一切しません。

監視中の状態変化

  • VRChat を起動した → 「監視中」フラグを ON に変更
  • VRChat を終了した → フラグが OFF になったことを検知
  • 終了を検知したら → 3秒待機後、ログの同期を開始
監視サイクル（3秒ごと）
VRChat 未起動 待機中
VRChat 起動検知 監視：ON
VRChat 終了検知 同期開始

3. ログ同期

VRC終了の瞬間に、更新分をコピー

VRChat が完全に終了したことを確認してから、ログファイルをチェックし、新規作成されたファイルや更新されたファイルのみをバックアップ先へ同期します。

同期対象ファイル

  • output_log_日付.txt （新規・更新分）

コピー元（VRChat ログ）

%APPDATA%\..\LocalLow\VRChat\VRChat\

保管先（Polaris バックアップ）

%LOCALAPPDATA%\CosmoArtsStore\STELLARECORD\app\Polaris\backup\
最新同期ログ
同期状態 最新
output_log_2026-02-28.txt ✓ 同期済
output_log_2026-02-27.txt ✓ 同期済

4. 自動復帰

何もしなくても、ずっと動き続けます

Windows Update などで PC が再起動されても、Polaris は自動で復帰します。設定ファイルやログ書き出しを行わないため、不具合が起きにくい設計です。

メンテナンスフリーの理由

  • Restart Manager 活用：Windows がアプリの再起動を管理
  • Zero Configuration：設定の不整合によるエラーを排除
  • Zero Log-Output：ディスク消費を抑え、クリーンに保持
システム安定性
Restart Manager 登録済
CPU / メモリ ほぼゼロ
ゲームのパフォーマンスには一切影響しません。
