### 調査対象: バックエンドロジックとデータベース

#### 発見した問題 #31
**ファイル:** `Alpheratz/src-tauri/src/scanner.rs`
**行番号:** 161 / 186
**該当コード:**
```rust
pub async fn compute_missing_phashes_bg(app: AppHandle) -> Result<(), String> {
...
        if let Ok(img) = image::open(&path) {
            let phash = hasher.hash_image(&img).to_base64();
```
**問題の内容:** `async` 関数内で同期的な `image::open` および `hash_image` をループ実行しています。
**影響範囲:** バックグラウンド処理中、Tokio ランタイムのスレッドが数秒〜数十秒占有され、他の非同期タスク（UIからのリクエスト等）のレスポンスが著しく低下します。
**推奨対応:** `spawn_blocking` を使用して計算処理を専用スレッドプールへ逃がす。

**指示:** その方針でOKです。

#### 発見した問題 #32
**ファイル:** `Alpheratz/src-tauri/src/scanner.rs`
**行番号:** 188-191
**該当コード:**
```rust
            let _ = conn.execute(
                "UPDATE photos SET phash = ?1 WHERE photo_filename = ?2",
                rusqlite::params![phash, filename],
            );
```
**問題の内容:** pHash の更新をループ内で 1 件ずつ個別に行っています。
**影響範囲:** SQLite のトランザクションが更新のたびに発生するため、大量の写真がある場合にディスク I/O 負荷が高まり、処理速度が低下します。
**推奨対応:** 一定件数（例: 50件）ごとにトランザクションをまとめてコミットする。

**指示:** その方針でOKです。
