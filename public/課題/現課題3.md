# Alpheratz 詳細調査報告書 第3弾（追加30項目・規定フォーマット版）

`flash-analyzer.md` の規定フォーマットに従い、`Alpheratz` プロジェクトの新たな 30 項目の課題（#61〜#90）を記述します。
これまでの報告書および `対応不要.md` と重複しない、アプリケーションの安定性、セキュリティ、保守性に関する項目を厳選しました。

### 調査対象: フロントエンド Hooks と UI コンポーネント

#### 発見した問題 #61
**ファイル:** `Alpheratz/src/hooks/useScan.ts`
**行番号:** 12 / 46
**該当コード:**
```typescript
        const init = async () => {
            await refreshSettings();
            await startScan();
        };
        init();
```
**問題の内容:** マウント時に自動で `startScan` が実行されますが、フォルダパスが未設定の場合でも強行されます。
**影響範囲:** 初回起動時など、設定が不完全な状態で不必要なエラーが発生し、UX を損なう可能性があります。
**推奨対応:** フォルダパスが有効（非 NULL かつ有効なパス）であることを確認してから実行するロジックの追加。

#### 発見した問題 #62
**ファイル:** `Alpheratz/src/hooks/useScan.ts`
**行番号:** 52
**該当コード:**
```typescript
unlistens.forEach(p => p.then(u => u()));
```
**問題の内容:** クリーンアップ時に `listen` （Promise）の解除を待機していません。
**影響範囲:** 高速な画面遷移（再レンダリング）が発生した場合、リスナーの登録と解除が交差し、メモリリークや二重登録の原因になります。
**推奨対応:** `Promise.all` 等で確実に全解除を待機するか、`listen` の解決を管理する設計への変更。

#### 発見した問題 #63
**ファイル:** `Alpheratz/src/hooks/useScan.ts`
**行番号:** 20, 36, 51, 60
**該当コード:**
```typescript
console.log("Scan completed received");
console.error("Scan error:", err);
```
**問題の内容:** `console.log/error` が直接使用されています。
**影響範囲:** 本番環境での不要なログ出力。保守時にログを追跡する仕組みが欠如しています。
**推奨対応:** バックエンドのログシステム（`utils.rs` 経由等）へ統一するか、環境に応じたログレベルの制御。

#### 発見した問題 #64
**ファイル:** `Alpheratz/src/hooks/usePhotos.ts`
**行番号:** 19
**該当コード:**
```typescript

```

**影響範囲:** エラーが発生しても UI 上で何も起きず、ユーザーが「読み込み中」のまま放置される可能性があります。
**推奨対応:** `addToast` 等を使用して、ユーザーにエラーを通知する。

#### 発見した問題 #65
**ファイル:** `Alpheratz/src/hooks/usePhotos.ts`

#### 発見した問題 #66
**ファイル:** `Alpheratz/src/components/PhotoGrid.tsx`


#### 発見した問題 #68

### 調査対象: バックエンド (Rust)

#### 発見した問題 #71
**ファイル:** `Alpheratz/src-tauri/src/lib.rs`
**行番号:** 75-93
**該当コード:**
```rust
pub async fn get_rotated_phashes(path: String) -> Result<Vec<String>, String> {
    let img = image::open(&path)... // 同期的な重い処理
    ...
```
**問題の内容:** 画像の読み込みと 3 回の回転計算（CPU 負荷大）を `async` ブリッジ上で直接実行しています。
**影響範囲:** 他の非同期タスク（通信やUIイベント）をブロックし、一時的にアプリがフリーズしたような挙動になります。
**推奨対応:** `tauri::async_runtime::spawn_blocking` を使用してスレッドを分離。

#### 発見した問題 #72
**ファイル:** `Alpheratz/src-tauri/src/lib.rs`
**行番号:** 109-111
**該当コード:**
```rust
"Alpheratz", 
"VR写真とワールド情報を紐付けるギャラリーツール"
```
**問題の内容:** STELLA RECORD 連携用の名称と説明がコマンド内に直接ハードコードされています。
**影響範囲:** 製品名の変更や多言語化に対応できません。
**推奨対応:** 定数化または `tauri.conf.json` のメタデータから取得するように改善。

**推奨対応:** `world_id` の形式（正規表現等）のチェック。

### 調査対象: インフラ・構成設定

#### 発見した問題 #81
**ファイル:** `Alpheratz/src-tauri/tauri.conf.json`
**行番号:** 22
**該当コード:**
```json
"csp": null,
```
**問題の内容:** CSP（コンテンツセキュリティポリシー）が未設定です。
**影響範囲:** XSS 等の脆弱性があった場合に、悪意のあるスクリプトの実行を制限できず、ローカルファイルへのアクセス権を奪われるリスクがあります。
**推奨対応:** 厳格な CSP の定義。

#### 発見した問題 #82
**ファイル:** `Alpheratz/src-tauri/tauri.conf.json`
**行番号:** 25
**該当コード:**
```json
"assetProtocol": { "scope": [ ... ] }
```
**問題の内容:** スキャン対象外かつ表示も行わないファイル形式（`.psd`, `.tiff` 等）までプロトコルのスコープに含まれています。
**影響範囲:** セキュリティ上の攻撃の窓口を不要に広げています。
**推奨対応:** サポートする画像形式（#50 のリスト）に限定。

#### 発見した問題 #83
