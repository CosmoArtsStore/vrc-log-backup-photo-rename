#### 発見した問題 #3

**ファイル:** `Alpheratz/src-tauri/src/scanner.rs`
**行番号:** 42, 64, 307-308

**該当コード:**
```rust
42:     let re_collect = Regex::new(r"(?i)VRChat_\d{4}-\d{2}-\d{2}_\d{2}-\d{2}-\d{2}\.\d{3}.*?\.(png|jpg|jpeg)").unwrap();
...
64:     let re_parse = Regex::new(r"VRChat_(\d{4}-\d{2}-\d{2})_(\d{2}-\d{2}-\d{2})\.(\d{3})").unwrap();
...
307:     let re_id = Regex::new(r"<vrc:WorldID>([^<]+)</vrc:WorldID>").unwrap();
308:     let re_name = Regex::new(r"<vrc:WorldDisplayName>([^<]+)</vrc:WorldDisplayName>").unwrap();
```

**問題の内容:** ループ内または頻繁に呼ばれる関数内で `Regex::new(...).unwrap()` が使用されている。正規表現の不適切な定義によるパニックのリスクがあり、また毎回コンパイルされるためパフォーマンスに影響する可能性がある。

**影響範囲:** アプリの安定性およびスキャン性能。

**推奨対応:** `once_cell` または `std::sync::LazyLock` (Rust 1.80+) を使用して静的に定義するか、初期化時に一度だけコンパイルするように変更する。

**修正時の注意点:** `StellaRecord` 側のパーサーでも同様の箇所があるため、併せて修正を推奨。

> 📝 **対応指示**:
> LazyLockで対応。

#### 発見した問題 #4

**ファイル:** `StellaRecord/stella_record_ui/src-tauri/src/analyze/mod.rs`
**行番号:** 179, 196, 203, 204, 210, 267, 268, 302, 327, 328, 349, 373

**該当コード:**
```rust
179:             let ts_str = caps.get(1).unwrap().as_str();
302:             let uid = caps.get(2).unwrap().as_str().to_string();
```

**問題の内容:** 正規表現の `captures` に対し、マッチングを前提とした `.unwrap()` が多用されている。パース対象のログ形式が将来変更された場合にパニックを引き起こす可能性がある。

**影響範囲:** ログのインポート・解析処理の堅牢性。

**推奨対応:** `.get().map(|m| m.as_str())` を使用し、安全に値を取得するか、適切なエラーハンドリングを行う。

**修正時の注意点:** ログパースはバッチ処理で行われるため、パニックによるアプリの強制終了は避けるべきである。

> 📝 **対応指示**:
> エラーハンドリングを実装する。パニックは絶対に避けること。
例：
// Before
let ts_str = caps.get(1).unwrap().as_str();

// After
let Some(ts_str) = caps.get(1).map(|m| m.as_str()) else {
    continue; // そのエントリはスキップしてログに残す
};
