# Alpheratz 詳細調査報告書 第4弾（追加30項目・規定フォーマット版）

`flash-analyzer.md` の規定フォーマットに従い、`Alpheratz` プロジェクトの新たな 30 項目の課題（#91〜#120）を記述します。
UI の細かな挙動、ビルドスクリプトの安全性、およびシステム全体の堅牢性に関する項目を集計しました。

---

### 調査対象: アプリケーションメインロジック (App.tsx)

#### 発見した問題 #91
**ファイル:** `Alpheratz/src/App.tsx`
**行番号:** 36-39
**該当コード:**
```typescript
  useEffect(() => {
    const t = setTimeout(() => setDebouncedQuery(searchQuery), 300);
    return () => clearTimeout(t);
  }, [searchQuery]);
```
**問題の内容:** 入力ごとに `debouncedQuery` を更新していますが、依存先の `usePhotos` でも `useEffect` が走るため、ステートの更新が連鎖します。
**推奨対応:** `useDeferredValue` の検討、またはデバウンス時間を定数化して管理。

#### 発見した問題 #92
**ファイル:** `Alpheratz/src/App.tsx`
**行番号:** 71
**該当コード:**
```typescript
    const selected = await open({ directory: true });
```
**問題の内容:** フォルダ選択ダイアログでエラーが発生した場合の `try...catch` がありません。
**影響範囲:** 特殊な権限エラー等でダイアログが開けない場合にアプリが沈黙します。
**推奨対応:** エラーハンドリングの追加。

#### 発見した問題 #93
**ファイル:** `Alpheratz/src/App.tsx`
**行番号:** 81
**該当コード:**
```typescript
const worldNameList = useMemo(() => Array.from(new Set(photos.map((p) => p.world_name || ""))).sort(), [photos]);
```
**問題の内容:** `photos` が大量（数千〜万）にある場合、再描画のたびに `Set` の生成と `sort` が走ります。
**推奨対応:** フィルタリング済みのリストではなく、バックエンド側でユニークなワールド名を取得するコマンドの検討。

#### 発見した問題 #94
**ファイル:** `Alpheratz/src/App.tsx`
**行番号:** 82
**該当コード:**
```typescript
const cellProps = useMemo(() => ({ data: photos, onSelect: onSelectPhoto, columnCount }), [photos, onSelectPhoto, columnCount]);
```
**問題の内容:** `columnCount` が依存配列に含まれているため、ウィンドウサイズを 1px 変えるごとに `cellProps` が再生成されます。
**推奨対応:** `columnCount` の変更が `PhotoCard` の内部ロジックに本当に必要か再考。

#### 発見した問題 #95
**ファイル:** `Alpheratz/src/App.tsx`
**行番号:** 148
**該当コード:**
```tsx
<div className="toast-icon">★</div>
```
**問題の内容:** トーストのアイコンが `★` 文字でハードコードされています。
**推奨対応:** メッセージの種類（success, error等）に応じたアイコンの切り替え。

### 調査対象: UI コンポーネント

#### 発見した問題 #96
**ファイル:** `Alpheratz/src/components/PhotoCard.tsx`
**行番号:** 24
**該当コード:**
```typescript
invoke<string>("create_thumbnail", { path: photo.photo_path })
```
**問題の内容:** `PhotoGrid` で表示される全カードが一斉に `invoke` を発行します。
**影響範囲:** 数百件の同時リクエストにより Tauri の IPC が渋滞し、UI レスポンスが低下する恐れがあります。
**推奨対応:** 表示範囲（Viewport）に入ったものから順次生成するレジーロードの強化。

**推奨対応:** 一定時間後の再試行ロジック、または「画像なし」用の代替アイコン表示。

### 調査対象: ビルドスクリプトとツール

#### 発見した問題 #104
**ファイル:** `Alpheratz/build-alpheratz.bat`
**行番号:** 8-11
**該当コード:**
```batch
taskkill /IM node.exe /F 2>nul
```
**問題の内容:** 全ての `node.exe` を強制終了させています。
**影響範囲:** 他の開発中のプロジェクト（StellaRecord等）のプロセスまで巻き添えで終了させます。
**推奨対応:** ポート番号を確認した上での特定プロセスの終了。

#### 発見した問題 #105
**ファイル:** `Alpheratz/build-alpheratz.bat`
**行番号:** 15
**該当コード:**
```batch
npm run build && npx tauri build
```
**問題の内容:** どちらかのコマンドが失敗しても、後の `pause` でしか気づけません。
**推奨対応:** エラーレベル（`%ERRORLEVEL%`）のチェックと、失敗時の明示的な通知。

#### 発見した問題 #106
**ファイル:** `Alpheratz/dev-alpheratz.bat`
**行番号:** 8-11
**問題の内容:** 課題 #104 と同様の広範囲なプロセス強制終了。
**推奨対応:** 開発効率を下げるため、本当に必要なプロセスのみを対象にする。

#### 発見した問題 #107
**ファイル:** `Alpheratz/package.json`
**行番号:** 11
**該当コード:**
```json
"tauri": "tauri"
```
**問題の内容:** `npx` 等を介さずに短縮名で呼ぶ想定になっています。
**推奨対応:** 環境に依存しないよう `tauri` を `npx tauri` 等に修正。

#### 発見した問題 #108
**ファイル:** `Alpheratz/src/App.tsx`
**行番号:** 72
**該当コード:**
```typescript
if (selected) { ... }
```
**問題の内容:** ユーザーがフォルダ選択をキャンセルした場合、`selected` は `null` になりますが、明示的な通知がありません。
**推奨対応:** キャンセル時は何もしない旨のログ出力、またはトースト表示。

#### 発見した問題 #109
**ファイル:** `Alpheratz/src/App.tsx`
**行番号:** 147
**該当コード:**
```tsx
key={t.id}
```
**問題の内容:** トーストのステート管理が単純な配列と `id` ベースです。
**推奨対応:** 重複メッセージの集約ロジックや、キューイング機能の検討。

### 調査対象: システム・設計

#### 発見した問題 #111
**ファイル:** `Alpheratz/src/index.css`
**問題の内容:** 変数定義などの基盤スタイルが空であり、`App.css` に全てのデザインが依存しています。
**推奨対応:** デザインシステムのトークン（色、余白）の整理。

#### 発見した問題 #112
**ファイル:** `Alpheratz/src-tauri/src/db.rs`
**問題の内容:** `PhotoRecord` 構造体のパス情報をそのままフロントエンドへ返却しています。
**影響範囲:** セキュリティ要件によっては、絶対パスの露出を控えるべき場合があります。
**推奨対応:** `convertFileSrc` 用の識別子や相対パスのみの返却を検討。

#### 発見した問題 #113
**ファイル:** `Alpheratz/src-tauri/src/utils.rs`
**行番号:** 19, 25
**問題の内容:** ログ出力関数（`log_info`, `log_err`）において、ファイル書き込みに失敗した場合の二次的手段がありません。
**推奨対応:** 標準出力へのフォールバック。

#### 発見した問題 #114
**ファイル:** `Alpheratz/src-tauri/src/scanner.rs`
**問題の内容:** ファイル名の非 UTF-8 文字が含まれる場合、`entry.path().file_name()` が取得できない可能性があります。
**推奨対応:** `OsStr` からの安全な変換と、エラー時の適切な無視/記録。

#### 発見した問題 #115
**ファイル:** `Alpheratz/src/hooks/useGridDimensions.ts`
**問題の内容:** リサイズイベントをそのままステート更新に反映させています。
**影響範囲:** 激しいリサイズ時に大量の再描画が発生し、特に `PhotoGrid` のような重いコンポーネントがある場合にボトルネックとなります。
**推奨対応:** `ResizeObserver` の活用か、リサイズ時のスロットリング。

#### 発見した問題 #116
**ファイル:** `Alpheratz/src/components/SettingsModal.tsx`
**問題の内容:** モーダルとしてのアクセシビリティ対応（フォーカストラップ等）が皆無です。
**推奨対応:** `aria-modal="true"` 等の属性追加、または Headless UI 系ライブラリの利用。

#### 発見した問題 #119
**ファイル:** `Alpheratz/src/App.tsx`
**行番号:** 61
**該当コード:**
```typescript
addToast("STELLA RECORD への連携を同期中...", "info");
```
**問題の内容:** プロセス完了後も「同期中」のトーストが残り続ける可能性があります。
**推奨対応:** 特定のトーストを ID で指定して消去または更新する仕組みの導入。

#### 発見した問題 #120
**ファイル:** `Alpheratz/src-tauri/Cargo.toml`
**行番号:** 27
**該当コード:**
```toml
windows = { version = "0.58", ... }
```
**問題の内容:** `windows` クレートのバージョンがやや古いです。
**推奨対応:** セキュリティや新機能の観点から最新安定版（0.59+）への更新検討。
