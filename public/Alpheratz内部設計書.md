# Alpheratz 内部設計書 (Internal Design Specification - Deep Dive Edition)

## 0. ドキュメントについて
本書は、VRChat 向け写真管理システム「Alpheratz」の完全な技術仕様を記述した内部設計書である。全てのソースコード、構成ファイル、およびインフラ設計に基づき、その構造と挙動を極めて詳細に解説する。

---

## 1. システム全体俯瞰

### 1.1 背景と目的
VRChat におけるスクリーンショット（思い出）は、単なる画像ファイル以上の価値を持つ。Alpheratz は、画像に埋め込まれたメタデータを解析し、STELLA RECORD の行動ログと照合することで、写真に「文脈（ワールド名、訪問日時、連れの友人）」を自動で付与し、高速な検索と閲覧を可能にすることを目的とする。

### 1.2 コア・アーキテクチャ
Tauri v2 を基盤とし、以下の3レイヤーで構成される：
1.  **Frontend Layer (React 19 / TypeScript)**: ユーザーインターフェース、仮想スクロール、クライアントサイド類似度計算。
2.  **IPC Layer (Tauri Bridge)**: 型安全なコマンド呼び出しと、バックグラウンド処理の進捗を伝えるイベントシステム。
3.  **Backend Layer (Rust / SQLite)**: ファイルシステム操作、バイナリチャンク解析、画像ハッシュ計算、DB 永続化。

## 2. バックエンド詳細詳解 (Rust Deep Dive)

### 2.1 メイン制御 (`main.rs`)
Windows ターミナルを表示せずに実行される「Windows サブシステム」環境下での堅牢性を確保。
- **致命的エラー捕捉**: `panic::set_hook` を通じ、Rust のパニックを Windows ネイティブの `MessageBoxW` へ変換。これにより、リリースビルドでのサイレントクラッシュを完全に排除している。
- **ログ記録**: クラッシュ直前に `info.log` への書き出しを保証。

### 2.2 コマンドインターフェース (`lib.rs`)
約 15 の Tauri コマンドを提供。
- **`initialize_scan`**: `tauri::async_runtime::spawn` を使用。OS レベルの非同期スレッドでスキャンを実行し、UI スレッドを 100% 解放することで、スキャン中もユーザーが以前の写真を閲覧可能なノンブロッキング設計を実現。
- **`get_rotated_phashes`**: `image` クレートの回転アルゴリズム (`rotate90`, `rotate180`, `rotate270`) をメモリエフェクティブに適用し、ハッシュ距離ベースの類似判定精度を向上。

### 2.3 高速スキャンエンジン (`scanner.rs`)
- **超並列性への対応**: Tokio ランタイムを活用しつつ、ファイル I/O は直列化。これは HDD 環境でのヘッドシーク負荷を抑制するための意図的な設計である。
- **PNG iTXt バイナリ解析**: 
    - ファイル全体を読み込まず、チャンクヘッダを逐次 `seek` する。
    - `iTXt` チャンクを特定後、`XML:com.adobe.xmp` キーワードを検索。
    - 内容物の XMP 文字列に対し、複数の正規表現 (`RE_ID`, `RE_NAME`) を適用。
- **データマージ**: 写真自体のメタデータと、`STELLA RECORD` DB の `world_visits` テーブルを、撮影日時をキーとして結合。不完全なメタデータを推論により補完する独自アルゴリズム。

### 2.4 データベース・デザイン (`db.rs`)
- **WAL モード**: 読み取りクエリ（写真閲覧）が、書き込みクエリ（スキャン中の INSERT）によってブロックされるのを防ぐ。
- **インデックス戦略**: `timestamp DESC` および `world_name` へのインデックスにより、数万枚の画像リストを一瞬で日付順にソートする。

## 3. フロントエンド詳細詳解 (Frontend & UI Deep Dive)

### 3.1 仮想化グリッドシステム (`PhotoGrid.tsx`)
`react-window` の `FixedSizeGrid` を極限までチューニング。
- **動的次元計算**: `useGridDimensions` がピクセル単位でカラム数を算出。
- **バッファリング**: 画面外の画像を上下3行分だけ保持し、スクロール時の描画遅延を隠蔽。

### 3.2 状態同期ロジック (Custom Hooks)
- **`useScan.ts`**: Tauri イベントリスナーを管理。`useEffect` のクリーンアップ節でリスナーを確実に解除し、メモリエラー（ゾンビリスナー）を防止。
- **`useMonthGroups.ts`**: 写真の日付から「年・月」の境界（インデックス）を算出し、スクロールバー横のフローティングインジケータへデータを提供。

### 3.3 類似画像検索アルゴリズム (`PhotoModal.tsx`)
- **ハミング距離計算**: 8ビットごとの `POPCNT` ルックアップテーブルを用意し、JavaScript のビット演算制限下で最高速のビットカウントを実現。
- **提案 UX**: 類似画像の中からメタデータを持つものを自動検索し、不明な写真の撮影ワールドを推定。ユーザーがワンクリックで確定できる「確証的 UI」。

## 4. インフラ・ビルド構成

### 4.1 NSIS インストーラー
`tauri.conf.json` とカスタム `.nsi` の組み合わせ。
- **レジストリ統合**: インストール時に `HKEY_CURRENT_USER\Software\CosmoArtsStore\STELLAProject\Alpheratz` へ `InstallLocation` を書き込む。これにより、アプリ自身や他アプリが構成ファイルを確実に特定できる。
- **Stargazer UI**: インストーラーのビットマップ画像をブランドカラーに統一。

### 4.2 依存関係とライセンス
- `lucide-react`: 軽量で見栄えの良いアイコンセット。
- `image_hasher`: 感知ハッシュ算出のコアライブラリ。

---

## 5. 技術的負債と将来の展望

### 5.1 既知の制約と対策
- **大量スキャン時のメモリ**: 画像デコード時のピークメモリ消費を抑えるため、スキャンバッチサイズを 50 枚に制限。
- **ファイルパスの非互換性**: パス区切り文字を Rust/JS 両側で正規化。

### 5.2 ロードマップ
1.  **AI 特徴量抽出**: SQLite の `photo_embeddings` テーブルを活用したセマンティック検索。
2.  **マルチフォルダ監視**: `notify` クレートを用いたリアルタイム写真検知。
