# テストチェックリスト生成スキル

ユーザーが持っているテスト項目リストと「ユーザーが不安に感じるポイント」を
組み合わせ、優先度付きのインタラクティブHTMLチェックリストを生成する。

## インプットの収集

次の情報を会話から読み取る（足りなければ質問する）：

1. **テスト項目リスト** — テキスト貼り付け・スプレッドシート・会話内容いずれでもよい
2. **アプリ/製品名** — ファイル名・プロセス名など具体的なものがあれば反映する
3. **ユーザーが不安に感じるポイント** — 自由記述でOK。なければ一般的な不安を推定する
4. **ファイルパス等の固有情報** — インストール先・設定ファイルの場所など（あれば）

## 優先度アサインのルール

ユーザーが挙げた「不安ポイント」を軸に各テスト項目の優先度を決める。

| 優先度 | 基準 |
|--------|------|
| 🔴 高 | ユーザーの不安ポイントに直結する。時間がなければここだけやれば最低限OK |
| 🟡 中 | 品質・安定性の確認。余裕があればやる |
| ⚪ 低 | エッジケース・開発者向け確認。余裕があれば |

**高優先度の目安は全体の20〜25%**。多すぎると意味がなくなる。

不安ポイントの例と対応する優先度判断：
- 「ウイルスじゃないか」→ 外部通信・ファイル改ざん確認を高に
- 「データが壊れないか」→ 書き込み先・削除有無確認を高に
- 「重くなる/悪影響が出る」→ CPU・メモリ・競合確認を高に
- 「ゴミが溜まる」→ 不要コピー・サイズ増加確認を高に
- 「常駐が不安」→ 落ちない・再起動できるを高に

## 言葉のルール

**テスターが一般ユーザーの場合（デフォルト）：**
- 技術用語を使わない
- 「アウトバウンド通信」→「インターネットにデータを送っていない」
- 「プロセス」→「起動中のアプリ」
- 「mutex」→ 説明不要（内部実装なので手順・期待結果に出さない）
- 「差分コピー」→「変化のないファイルは上書きしない」
- 確認手順の欄は「〇〇を開いて〇〇を確認する」レベルの粒度で書く

**テスターが開発者の場合：** 技術用語そのまま使用してOK。

## 出力フォーマット

`assets/template.html` を必ず読んでから生成する。
テンプレートの構造・スタイル・JS構成を維持しつつ、以下の変数を差し替える：

- `{{APP_NAME}}` — アプリ名
- `{{PATHS_HTML}}` — パス情報のHTMLブロック（不要なら空文字）
- `{{TESTS_JS}}` — テスト項目の配列（JS）
- `{{AX_JS}}` — 不安カテゴリの配列（JS）

### テスト項目のデータ構造

```js
{
  id: "3.1.1",          // 元の番号をそのまま使う（なければ連番）
  cat: "③プロセス監視",  // 大項目（絵文字番号+名前）
  priority: "高",        // "高" | "中" | "低"
  title: "...",          // 一言で何をテストするか（30字以内推奨）
  desc: "...",           // 確認手順（改行は \n）
  expect: "...",         // こうなればOKの期待結果
  anxiety: ["game"],     // 対応する不安カテゴリのID配列
  isNew: true,           // 追加項目の場合のみ true（それ以外は省略）
}
```

### 不安カテゴリのデータ構造

```js
{ id: "game", icon: "🎮", label: "ゲームに悪影響がない", color: "#00d2ff" }
```

不安カテゴリIDとラベルはプロジェクトに合わせて自由に設定してよい。

## 追加項目の自動検討

テスト項目リストを見て、不安ポイントに対応する項目が不足している場合は
`isNew: true` で補完項目を生成する。特に漏れやすいもの：

- 外部通信の有無（ウイルス不安がある場合）
- 元ファイルの削除・改ざん有無（データ安全不安がある場合）
- 長時間稼働時のメモリ増加（常駐不安がある場合）

## ファイルの出力

`/mnt/user-data/outputs/テスト優先度チェックリスト.html` に保存し、
`present_files` ツールでユーザーに渡す。
