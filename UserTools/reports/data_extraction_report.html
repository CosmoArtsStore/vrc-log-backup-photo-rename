STANDARD MODE

00

セッション基本メタデータ

Standard
セッション継続時間 (Session Uptime)

この項目は、VRChatアプリケーションが起動してから「現在」または「異常終了する直前」までの経過時間を秒単位で記録するものです。単なる時刻の差分ではなく、バックグラウンドでの待機時間も含めた「VR空間に接続し続けていた総実時間」を定義します。

ログには一定間隔で「現在の累積時間」が書き込まれるため、万が一ゲームがクラッシュして「終了ログ」が残らなかった場合でも、直前の記録からほぼ正確な滞在時間を復元することが可能です。

  • 週次・月次の「総プレイ時間」のヒートマップ作成。
  • 「何時間連続でプレイするとFPSが低下し始めるか」というメモリリーク診断の基準軸。
  • フレンドとの「遭遇時間」を算出し、親密度をスコア化するアルゴリズムへの利用。

03

ユーザー設定スナップショット

Standard
オーディオ・グラフィックス設定

起動時にVRChat側で保持されている、あらゆる「設定値」をスナップショットとして記録します。これには、マイクボリュームやマスターボリューム、BGMの音量バランスといったオーディオ設定が含まれます。例えば、「今日はなぜか声が小さいと言われた」際、ログを振り返れば「マイク設定が不意に20%に下がっていた」といった事実がすぐに判明します。

また、OSC（Open Sound Control）が有効になっているか、どのポートを使用しているかも記録します。外部ツール（OyasumiVR等）との連携が上手くいかない際の切り分けに役立つほか、グラフィックスのAA（アンチエイリアス）設定やMirror解像度の設定値も網羅するため、負荷テスト時の「基準設定」を後から参照する際にも重宝します。

  • 「マイクの切り忘れ」や「設定ミス」を過去のログから確認・自己診断。
  • OSCポートの重複や衝突のトラブルシューティング。
  • 「一番綺麗に見える設定」と「一番軽い設定」の数値的な差を分析。

04

ソーシャル・インタラクション

Standard
受取通知と自身のリアクション履歴

フレンドから届いた「Boop（つつき）」「Invite（招待）」「friendRequest（申請）」の履歴を、誰から届いたか、そしてその際に添えられていた「カスタムメッセージ」の内容も含めて保存します。VRChatの画面内では流れて消えてしまうこれらのメッセージを、テキストとして永続化することで、後から「あの時誰に誘われたか」を確実に振り返ることができます。

それだけでなく、自身がその通知に対して行ったアクション（承諾したのか、無視したのか）も記録されます。また、自身のマイクのミュート切替や、ボリューム調整などの「自身のデバイス操作」も時系列で追えるため、プレイ中の自分の振る舞いをメタ的に把握し、ボイスチャットのトラブル防止やマナー向上にも繋げることが可能です。

  • 届いた Invite メッセージの検索（「今からイベントです」等のキーワード検索）。
  • 自分が誰からの Boop に反応したかの相関分析。
  • マイクのミュート忘れによる「事故」がどのタイミングで起きたかの事後検証。

05

ワールド探索・滞在ログ

Standard
入退室履歴と恒久ユーザーID (usr_ID)

インスタンス内に存在する他のユーザーの「入室」と「退室」を、表示名（Display Name）だけでなく、バックエンドで管理されているユニークな「usr_ID」と共に記録します。表示名はユーザーが自由に変更可能ですが、usr_IDは不変であるため、名前を変えたフレンドの追跡や、過去に会った人物の特定が確実に行えます。

このデータには「どのインスタンスIDで会ったか」が紐付くため、「特定のイベントワールドで出会った人リスト」を後から自動生成することが可能です。また、退室ログがあることで、その人がインスタンスに何分間滞在していたかも算出できます。

  • 「会ったことのあるユーザー名」の全文検索機能の実装。
  • フレンドが名前を変更した際の自動通知・更新システム。
  • 大人数が集まるイベントでの「参加者名簿」の自動作成。
Technical Mapping

08

Udon ギミック・外部連携

Standard
Udonスクリプト動作とOSCサービス検出

ワールド内の様々な仕掛けを制御する「Udon」スクリプトや、主要なサードパーティ製ギミック（USharpVideo, YamaStream等）の動作ログを抽出します。これにより、特定のワールドで「ビデオプレイヤーがなぜか動かなかった」といった事象の際、初期化の失敗やURL読み込みのタイムアウトがいつ起きたのかを具体的に特定できます。

また、あなたのPCで動作しているOSCサービス（OyasumiVR等）をVRChatが自動検出した際の記録を取得します。標準モードでは「どのサービスがいつ起動・検出されたか」という生存確認レベルの情報を収集します。なお、通信ポート番号や詳細なネットワーク属性については、トラブルシューティング用の高度な情報であるため「開発者モード」に限定して記録する運用とします。

  • ビデオプレイヤーの読み込み失敗原因（URLエラー/yt-dlpエラー等）の特定。
  • 外部連携ツール（OyasumiVR等）の自動起動・認識の成功確認。
  • 特定のワールドギミックが自分の環境で正しく初期化されているかの証拠。
Technical Mapping
[Behaviour] YamaStream: Initialization started. [Behaviour] Found OSC service: OyasumiVR
Field: udon_event_tag
Field: osc_service_name
Freq: Initialization / Event
11

自己アバター・ライフサイクル

Standard
自分のアバター変更ログとロード解析

他人のアバター詳細はプライバシー保護のため一切記録しませんが、あなた自身がアバターを着替えた履歴は「セルフ・ファッションログ」として詳細に記録します。アバター名、ID、およびデータサイズ（MB）を保持することで、過去の自分の姿を振り返る重要なライフログとなります。

また、着替え完了までにかかった実時間を秒単位で算出します。このデータは、特定のワールドでの着替えの重さを客観的に評価する際や、通信環境の安定性をアバターロードという側面から診断するのに非常に有用です。

  • 自分自身のアバター着替え履歴（自分史）の作成。
  • アバターごとのロード時間・成功率の分析。
  • 特定アバター使用時のパフォーマンス特性の事後確認。

12

ビデオプレイヤー記録

Standard
動画再生URLと同期ステータス

ワールド内で再生された、あらゆる動画（YouTube, Twitch等）の「URL」を時系列で保存します。再生開始、一時停止、終了といったイベントを、再生位置（タイムスタンプ）と共に記録するため、後から「あのワールドで流れていたあの曲は何だったか」を確実に、かつ即座にリスト形式で振り返ることができます。

また、AVProプレイヤーが動画を解析（Resolve）した際の技術的なログも保持します。これにより、特定の動画が「エラーで再生できなかった」理由が、URLの存否なのか、それともUnityエンジンのコーデックエラーなのかを切り分けられます。プライバシー保護のため、リクエスタ（誰が流したか）の個人名は破棄し、純粋なメディア履歴のみを抽出します。

  • ワールド内での「プレイリスト」自動生成。
  • 動画読み込みエラー時の「原因特定」と「回避策の検討」。
  • 複数のビデオプレイヤーが混在するワールドでの再生優先度の動作確認。

Field: player_status

13

写真・メディアファイル連携

Standard
写真の絶対パスと解像度データ

VRChat内でカメラのシャッターを切った瞬間、あなたのPCのハードディスク（あるいはSSD）の「どこに」その写真が保存されたかを、フルパスで完全に記録します。これがあることで、後から外部のログビューア上で写真をクリックした際に、直接ファイルを開いたり、ギャラリーを自動生成したりといった強力な「ファイル管理連携」が可能になります。

また、その写真が「どのワールドで」「誰と一緒にいた時に」撮られたものかを、同時刻の「入退室ログ」や「ワールド滞在ログ」と自動的に紐付けることが本プロジェクトの究極の目的の一つです。撮影時の解像度（4K判定等）も保持するため、高画質な思い出写真だけを自動でピックアップするような抽出も容易になります。

  • 撮った写真から「その時一緒にいた人リスト」を自動生成。
  • 特定のワールドで撮った写真だけを抽出するデジタルアルバム機能。
  • 写真ファイルの場所を忘れても、ログから即座にファイルパスを特定。

ソーシャル深化 & 運営フラグ

Standard
グループ参加・フレンド更新 (Cat 14, 15, 16)

新しく所属したVRChatグループや、その中でのランク変更、さらには「現在オンラインのフレンド数」の推移などを記録します。これらは、あなたのVRChat内での「社会的な繋がり」の広がりを客観的な数値として残すものです。いつの間にか増えていたフレンドや、以前所属していたグループの履歴を整理する際に役立ちます。

また、公式が配信する「今日のメッセージ」や、サーバーサイドからのグローバルな制限フラグに関するログも抽出します。これにより、VRChat公式がどのような告知やアップデートをリアルタイムに反映させているのかを、システム側の視点から把握することが可能です。自身のステータス（Green/Blue等の表示）の変更履歴も保持されます。

  • フレンド数の増加グラフ의作成（ソーシャル成長記録）。
  • 自分がどの時間帯に「ステータス」を切り替えていたかの自己分析。
  • 各グループ活動の時期の整理・自分史の作成。

DEVELOPER MODE

01

ハードウェア・ランタイム詳細

Developer
PCスペックとVRデバイス構成

このデータ群は、VRChatが起動した瞬間に読み取られる「あなたのPCの体力」の証明です。CPUの型番からGPU（グラフィックカード）の名称、VRAM（ビデオメモリ）の容量、さらにはOSのバージョンまでを完全に網羅します。これにより、特定のワールドでの不調が「グラボのメモリ不足」なのか「CPUの世代の古さ」なのかを客観的に切り分けることが可能になります。

また、使用しているHMD（Meta Quest 3, Valve Index等）や、左右それぞれのコントローラーの種類も個別に記録されます。冬場に電池切れでコントローラーが切断された際や、トラッキング方式の変更（Oculus Native / SteamVR）による遅延の差などを後志で分析する際に、この機器情報のスナップショットが極めて重要な意味を持ちます。

  • 自分のPCスペックの変遷（パーツ交換履歴）をログとして自動保存。
  • 「Quest 2使用時」と「Quest 3使用時」でのFPS安定度の自動比較レポート。
  • 特定のVR機器だけで発生する「マイク認識エラー」の特定。

02

インフラ・ネットワーク接続

Developer
Photon / EOS 通信品質統計

VRChatのマルチプレイヤー通信の心臓部である「Photonサーバー」や「EOS（Epic Online Services）」との通信状態を、ミクロな単位で記録します。単にネットが繋がっているかだけでなく、「どのリージョンのサーバーに繋がったか（Japan / US / EU）」「パケットが何個の束（Bunches）で送られたか」といった、低レイヤーの通信効率を可視化します。

特に「Disconnect (Reason: 5)」などの切断理由コードを取得することで、それが「サーバー側のダウン」なのか「自分の家の回線のタイムアウト」なのかを100%特定できます。これは、夜間のゴールデンタイムに発生しやすい「突発的なサーバー追い出し」の証拠として、あるいは回線業者への相談資料としても機能するほど具体的なデータです。

  • 「Japan リージョン」と「US リージョン」での平均 Ping 特性の比較。
  • 回線が切れやすい時間帯の特定による、ルーターの自動再起動タイミングの検討。
  • サーバーメンテナンスや障害発生時の「ダウンタイム」の正確な記録。
Technical Mapping

Field: photon_region_code
Field: packet_bunches_count
Source: Network Bridge

06

パフォーマンス統計 (Heartbeat)

Developer
JSON Performance Stats (毎分統計)

VRChatエンジンが1分おきに自動出力する「Performance Stats」ブロックから、FPSやPingの平均/最小/最大値、および描画負荷の具体的な数値を抽出します。単なる平均値だけでなく「最小FPS」を取得することで、一瞬のカクつき（スタッター）が発生したタイミングを特定できます。

特筆すべきは「周囲のアバター負荷」の詳細な内訳です。ExcellentからVeryPoorまでの各ランクのアバターが周囲に何人いたのか、またポリゴン数やボーン数、マテリアル数の合計が毎分記録されます。これにより、「FPSが落ちた原因は、VeryPoorアバターが急に5人増えたからだ」といった因果関係を客観的に証明できます。

  • 特定ワールドでの平均FPSを算出し、自分のPCスペックでの「快適度」をランキング化。
  • 「Poor以下のアバターを非表示にした場合」と「そうでない場合」の負荷軽減率の比較。
  • ネットワーク遅延（Ping）の推移をグラフ化し、プロバイダやWi-Fiの安定性を診断。

09

例外・デバッグログ

Developer
エンジン例外エラーとAPIレスポンス

VRChatの動作中に発生した「NullReferenceException（ヌルリポ）」や「UdonVMException」といった、プログラムレベルの致命的なエラーを完全なスタックトレースと共に記録します。これは「ワールドが重くなった」「ギミックが反応しなくなった」といった曖昧な不具合の背後にある、論理的な故障原因を突き止めるための「決定的な証拠」となります。

また、VRChatのバックエンドAPIとの通信で発生したエラー（404 Not Found / 503 Service Unavailable等）も収集します。アバターが表示されない、フレンドリストが読み込めないといったトラブルが「公式サーバーの不調」によるものなのか、それとも「特定の通信の失敗」によるものなのかを、生データに基づいて判断できます。

  • ワールド内の特定の椅子に座るとエラーが出る、といった「踏むと危ない箇所」の判別。
  • VRChat公式サーバーの障害発生時刻の正確な記録。
  • アバターロード失敗時の「シェーダーコンパイルエラー」等の詳細原因特定。

10

高度な統計・ランタイム指標

Developer
入力遅延とフレームタイムの内訳

単なるFPS数値の裏側にある、非常に低レベルな「処理時間」の内訳を抽出します。自分の手が動いてから画面に反映されるまでの「input_latency」や、CPUとGPUそれぞれが1フレームの描画に何ミリ秒費やしたか（cpu_frame_time / gpu_frame_time）を記録し、PCのボトルネックがどこにあるかを、これ以上ないほど正確に定量化します。

また、あなた自身に付与されている「DeveloperTrustLevel（信託ランク）」や、内部的な機能制限フラグの状態もスナップショットとして保持します。これにより、アップデート後に自分のアカウントに新しい機能が反映されたか、あるいは内部的なデバッグフラグが予期せずオンになっていないかなどを、システム管理者レベルの視点で監視できます。

  • 「VRコントローラーの反応が数ミリ秒遅い」といった違和感の数値的な裏付け。
  • CPU性能が限界なのか、グラボ性能が限界なのかの「ボトルネック判定」。
  • 自分のアカウントに適用されている隠しテスト機能（ベータ機能）の確認。

18

インスタンス・ポータル内部構造

Developer
ポータルの生成・消滅と同期データ量

ワールド内に動的に設置される「インスタンス・ポータル」の全挙動を、生成から消滅までミクロに追跡します。誰かが「別のワールドへの門」を開いた瞬間、それが手動配置だったのか、それともポータル生成がシステム的に拒否されたのかといった、ワールド内の物理的な変化をデータとして記録します。

さらに、インスタンス入室時の「メタデータ同期」にかかったパケットサイズや、シーン全体がメモリに展開されるまでの「詳細な工程（instantiateScene, WarmShaders等）」を秒単位で抽出します。これにより、「このワールドが重いのはシーンのロードが遅いからではなく、シェーダーのコンパイルが長いからだ」といった技術的な真実が見えてきます。

  • ロード時間の異常（1分以上かかる等）をアラートとして検出。
  • ワールド制作者向けの、ロード時間改善のためのデバッグ資料。
  • ポータルが開いた時間の記録から、イベント進行のタイムラインを自動構成。
